# 업무보드 리디자인 설계안

작성일: 2026-03-03
대상: 1인 VC 백오피스 관리자

---

## 설계 목표

1. **누락 제로** — 세세한 업무까지 놓치지 않는 구조
2. **연계성 파악** — 업무가 어떤 펀드·워크플로·컴플라이언스와 연결되는지 클릭 없이 보임
3. **1인 최고 효율** — 클릭 최소화, 인지 부하 최소화
4. **폭주 대응** — 10개 조합 총회가 동시에 돌아도 피로하지 않은 구조

---

## 핵심 문제 정의

### "전부 보여주면 피곤하고, 숨기면 누락된다"

이 모순을 해결하는 열쇠:
- **정상인 건 작게, 문제인 건 크게** (이상 감지 원칙)
- **맥락은 항상 보이되, 상세는 필요할 때만** (카드에 펀드명·마감 표시, 메모는 클릭 시)
- **폭주 시 자동 축약** (업무 10건 이하면 전부 펼침, 30건 넘으면 그룹 접힘)

---

## 유지하는 것

- **아이젠하워 4분면** — 우선순위 프레임으로 유효. 버리지 않음
- **캘린더 뷰** — 일정 파악용으로 유지
- **파이프라인 뷰** — 시간 흐름 파악용으로 유지
- **드래그앤드롭** — 분면 간 이동
- **워크플로 그룹 카드** — 이미 미래 단계 숨김, 현재 단계 표시 중. 유지
- **필터 패널** — 상태, 검색, 기한, 대상, 카테고리

## 추가/변경하는 것

### 1. 상단 요약 바 (신규)

대시보드에서 이관되는 업무 현황 요약. **항상 보임, 클릭 불필요.**

```
┌────────┬────────┬────────┬────────┬──────────────────────────────┐
│🔴 지연  │🟠 오늘  │🟡 이번주│✅ 완료  │ 예상 2h40m  ████████░░ 65%  │
│   2건   │   5건   │   8건   │   3건   │ 업무 점수 ██░░ 72           │
└────────┴────────┴────────┴────────┴──────────────────────────────┘
```

- 각 숫자 클릭 → 해당 기한 필터 즉시 적용 (한 번 클릭으로 필터)
- **업무 점수**: 대시보드 건강 점수 시스템과 연동. 지연·방치가 많으면 낮아짐
- **프로그레스 바**: (오늘 완료 / 오늘 전체) — 하루의 진척 시각화

### 2. 태스크 카드 정보 강화 (변경)

현재 카드에 보이는 것: 제목, 긴급도 뱃지/카테고리(둘 중 하나), 마감일, 예상시간

**변경 후 카드에 항상 보이는 것:**

```
┌────────────────────────────────────────────────────────────┐
│ ≡ ☐  분기보고 초안 작성                    ✓완료  수정  × │
│      1호펀드 · LP보고 · D+1 지연 · 1h · ⚠ 방치 3일       │
└────────────────────────────────────────────────────────────┘
```

| 정보 | 현재 | 변경 후 |
|------|------|---------|
| 제목 | ✅ 보임 | ✅ 유지 |
| 긴급도 뱃지 | ✅ 보임 (카테고리와 택1) | ✅ 항상 보임 |
| 카테고리 | ⚠ 긴급도와 택1 | ✅ 항상 보임 (동시 표시) |
| 펀드/GP명 | ⚠ 카드에 안 보임 (모달에서만) | ✅ 항상 보임 |
| 마감일 | ✅ 보임 | ✅ 유지 + D-day/D+day 표시 |
| 예상시간 | ✅ 보임 | ✅ 유지 |
| 워크플로 연결 | ⚠ 그룹카드 안에서만 | ✅ 워크플로명 뱃지 |
| **방치 경고** | ❌ 없음 | ✅ 신규: 생성 후 N일 미처리 시 경고 |
| **컴플라이언스 연결** | ❌ 없음 | ✅ 신규: 컴플 의무 연결 시 뱃지 |

핵심: **한 줄 추가**로 펀드·카테고리·마감·방치 상태를 전부 표시. 카드 높이는 기존 대비 ~15px만 증가.

### 3. 방치 경고 시스템 (신규)

**"잊혀진 업무"를 자동 감지:**

| 조건 | 경고 |
|------|------|
| 생성 후 3일 이상 상태 변경 없음 (pending 유지) | ⚠ 방치 3일 |
| 생성 후 7일 이상 | 🔴 방치 7일 (강조) |
| 마감일 없는 업무가 5일 이상 | ⚪ 기한 미설정 5일 |

- 카드 메타 행에 경고 뱃지 자동 표시
- 상단 요약 바의 "업무 점수"에 반영 (방치 건수 → 감점)
- **클릭 없이 보임** — 카드를 스캔하다가 ⚠ 뱃지가 눈에 들어옴

### 4. 스마트 접기 (폭주 대응, 신규)

**업무가 많아질 때 자동으로 밀도를 조절:**

| 분면 내 업무 수 | 표시 방식 |
|----------------|----------|
| 0~8건 | 전부 펼침 (현재와 동일) |
| 9~15건 | 지연+오늘 업무만 펼침, 나머지 접힘 ("이번 주 외 N건 ▶") |
| 16건 이상 | 지연+오늘만 펼침 + 워크플로 그룹은 프로그레스 바만 표시 |

**워크플로 그룹 스마트 접기:**

| 상태 | 표시 |
|------|------|
| 현재 단계가 지연/오늘 마감 | 펼침 (현재 단계 카드 보임) |
| 현재 단계가 이번 주 이후 | **접힘** (1줄 요약만: "투자심의-A사 3/7 ████░░░ 43%") |
| 전체 완료 | 숨김 또는 최소 표시 |

→ 10개 조합 총회 워크플로가 동시에 돌아도:
- 오늘 처리할 단계가 있는 3개만 펼침
- 아직 안 온 7개는 접힌 1줄 프로그레스 바
- 화면에 실제 보이는 건 "오늘 할 것 + 지연" 위주

**수동 오버라이드**: 접힌 그룹 클릭하면 펼쳐짐. 자동 접기는 기본값일 뿐, 강제가 아님.

### 5. 완료 업무 인라인 (대시보드에서 이관)

현재 업무보드에는 완료 업무를 보려면 상태 필터를 "완료"로 바꿔야 함.

**변경**: 보드 하단에 "오늘 완료" 접기식 섹션 추가.

```
├─────────────────────────────────────────────────────────────┤
│ ✅ 오늘 완료 (3건)                                    ▼ 펼치기│
│  ✓ NAV 확인  15m  │  ✓ LP 연락  10m  │  ✓ 메일 발송  5m   │
│                                              [되돌리기] 각 항목│
└─────────────────────────────────────────────────────────────────┘
```

- 기본 접힘 (건수만 보임)
- 펼치면 완료 업무 + 소요시간 + 되돌리기 버튼

### 6. 사이드 패널 (모달 대체)

현재 업무 수정/상세가 **모달**로 열림 → 보드가 가려짐 → 맥락 단절.

**변경**: 우측 사이드 패널로 전환.

```
┌─────────────────────────────────┬──────────────────────────┐
│ 보드 (좌, 그대로 보임)          │ 상세 패널 (우, 슬라이드) │
│                                 │                          │
│  Q1 긴급·중요                   │ 📋 분기보고 초안         │
│  ┌───────────────────┐         │                          │
│  │ ● 분기보고 초안 ←선택│       │ 마감: 3/2 (D+1)         │
│  │   1호펀드 D+1     │         │ 펀드: 1호 펀드           │
│  └───────────────────┘         │ 카테고리: LP보고         │
│  ┌───────────────────┐         │ 예상: 1h                │
│  │ ○ VICS 보고       │         │ 메모: LP 실적 데이터...  │
│  │   공통 오늘       │         │                          │
│  └───────────────────┘         │ 첨부: Q1_초안.xlsx       │
│  ...                            │                          │
│                                 │ [완료] [저장] [삭제]     │
│  Q2 중요·비긴급                 │                          │
│  ...                            │                          │
└─────────────────────────────────┴──────────────────────────┘
```

- 카드 클릭 → 우측 패널 슬라이드 (보드는 그대로 보임)
- 패널에서 바로 편집·완료·삭제 가능
- ESC 또는 패널 외부 클릭으로 닫기
- **보드를 보면서 상세를 보는 게 핵심** — 맥락 유지

## 백엔드 변경

### 기존 API 확장

`GET /api/tasks/board` 응답에 summary 추가:

```json
{
  "summary": {
    "overdue_count": 2,
    "today_count": 5,
    "this_week_count": 8,
    "completed_today_count": 3,
    "total_pending_count": 24,
    "total_estimated_minutes": 160,
    "completed_estimated_minutes": 65,
    "stale_count": 3
  },
  "Q1": [...],
  "Q2": [...],
  "Q3": [...],
  "Q4": [...]
}
```

### Task 모델에 방치 감지 필드

별도 필드 추가 불필요. 프론트에서 계산:
```
방치일수 = (현재 시각 - task.updated_at 또는 task.created_at) / (1일)
조건: status === 'pending' && 방치일수 >= 3
```

또는 백엔드에서 `stale_days` 계산 필드를 응답에 포함:
```json
{
  "id": 42,
  "title": "분기보고 초안",
  "stale_days": 5,
  ...
}
```
