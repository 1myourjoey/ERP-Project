# Phase 28_1: 글로벌 조합 통제실(Capital Dashboard) 개편 및 실행 기능(`FundDetailPage`) 완벽 이관

> **Priority:** P0 (Information Architecture & VC Operations UX Paradigm Shift)

---

## 🚨 핵심 기획 배경 (Why we do this)
현재의 `FundOperationsPage.tsx(관리 탭 > 조합운영)`는 이름은 전역 관리 탭이지만, 실제 들어가면 상단에서 "개별 펀드 1개"를 고른 뒤 해당 펀드의 특정 실무(LP 양도, 분배금 지급, 총회 등록)를 조작하는 기형적인 구조를 지니고 있습니다. 이는 **"상황실에 들어갔더니 결재 서류 1장을 들이미는 꼴"**로, VC 현업의 운영 멘탈 모델에 위배됩니다.

따라서 코덱스가 앞선 Phase 28에서 5-Tab으로 예쁘게 쪼개어 완성한 `FundDetailPage.tsx(개별 펀드 상세뷰)`에 **펀드와 관련된 모든 '실행(Action)' 기능을 완벽하게 100% 몰아주고**, 기존의 전역 관리 탭은 **비행기 계기판 같은 완벽한 뷰어(글로벌 자본 관제탑)**로 탈바꿈시키는 거대한 정보구조(IA) 재배치를 지시합니다.

---

## Part 1. `FundOperationsPage` 내 모든 실행 기능의 완벽한 짐싸기(Migration)

기존 `FundOperationsPage.tsx`가 품고 있던 핵심 비즈니스 로직(양수양도, 분배, 총회) 컴포넌트와 API 통신 코드 전체를 완벽하게 떠서, Phase 28에서 구축한 `FundDetailPage.tsx` 의 해당 탭(Tab) 안으로 100% 흡수/이사시켜야 합니다.

| 소거할 기존 기능 (From `FundOperations`) | 이사 보낼 최종 목적지 방 (To `FundDetail` Tabs) |
|---|---|
| **LP 양수양도 (Transfer)** 모달 및 목록 | **`Tab 3. 자본 및 LP 현황`** 내 하단 섹션으로 병합 |
| **분배 (Distribution)** 내역 관리 표 및 지급 모달 | **`Tab 3. 자본 및 LP 현황`** 내 하단 센션으로 병합 (자본 탭 1개에서 출자/양도/분배 3가지 돈맥경화를 모두 통제) |
| **조합 총회 일정 (Assembly)** 등록 및 현황 | **`Tab 5. 규약 및 컴플라이언스`** 내부로 이사 (총회는 법적 의무 사항이므로 컴플라이언스 탭이 적합) |

> **⚠️ 기능 무결성 경고:** 이사하는 과정에서 버튼(`[+양수양도]`, `[+배분 추가]`)이 사라지거나 백엔드 호출 Mutate 가 깨지지 않도록(`queryClient.invalidates`), 코드 복붙 시 Props 타겟팅(`fundId` 매핑)에 극도로 주의하십시오.

## Part 2. 글로벌 `FundOperationsPage.tsx` 의 진정한 상황실(Control Tower) 대모험

이제 모든 짐(기능)을 덜어내서 텅 빈 `FundOperationsPage.tsx` 화면을, 단 한 번의 클릭이나 드롭다운 변경 없이 **운용 중인 모든 펀드의 자본 상태를 한눈에 볼 수 있는 마스터 엑셀 대시보드(Master Capital Grid)**로 싹 다 갈아엎습니다.

### 2-A. 마스터 테이블 (Global Fund Summary Grid) 규격
새 조합 운영 페이지에 진입하면, 우리 회사가 보유한 전체 펀드 리스트가 아래와 같은 표 형태로 넓게 깔려야 합니다. (어떤 데이터 생성/수정/삭제도 불가능한 Read-Only 관제탑)

| 펀드명 | 구분 | 결성일 | 총 약정액 | 누적 출자 지시액 (납입률%) | 현재 미납 잔액 | 누적 분배액 (분배율%) | 상태 |
|---|---|---|---|---|---|---|---|
| 밸류체인 1호 | 블라인드 | 24-01-01 | 200억 | 100억 (50%) | **0원** | 0원 (0%) | 운용중 |
| 청년창업 2호 | 프로젝트 | 25-05-15 | 500억 | 480억 (96%) | **20억** (⚠️) | 150억 (30%) | 운용중 |
| ... | ... | ... | ... | ... | ... | ... | ... |
| **우리 회사 총괄** | | **Total** | **700억** | **580억** (통합 82%) | **20억** | **150억** (통합 21%) | |

1. **컬럼별 계산 로직:** 최신 데이터(React Query 캐싱)를 끌어와 `(누적 출자액 / 총 약정액) * 100` 의 퍼센트와 포맷팅된 KRW를 함께 보여줍니다.
2. **미납 경고 UI (`⚠️`):** 합산된 미납 잔액이 0보다 큰(1원이라도 비는) 펀드의 경우, 글씨를 붉은색 굵게(`text-red-600 font-bold`) 처리하고 뱃지를 달아주어 상황실 근무자가 즉각적으로 이슈를 판별할 수 있게 합니다.
3. **Table Footer 합산:** 가장 밑바닥에는 전체 운영 펀드의 AUM(총약정), 누적투자, 합산미납을 합친 글로벌 Total 수치가 엑셀처럼 렌더되어야 합니다.
4. **딥링크(Deep Link) 라우팅:** 표의 로우(행) 혹은 특정 펀드 이름을 클릭하면, 그 즉시 **해당 펀드의 `FundDetailPage.tsx` 로 순간 이동(Navigate)** 되어 현업 담당자가 미수/지연 내역을 조치할 수 있게 합니다.

## Acceptance Criteria

- [ ] AC-01: 사용자가 GNB(메뉴)에서 [관리 > 조합운영] 탭을 눌렀을 때, 펀드를 선택하는 행위 없이 **운용 중인 모든 펀드 목록과 총 자본 합계(AUM)가 엑셀 데이터 통계표처럼 한 번에** 보여야 한다.
- [ ] AC-02: 표에서 특정 펀드에 미납 금액이 발생했을 시 눈에 확 띄는 컬러/뱃지 형태의 시각적 알람(Alert)이 제공되어야 한다.
- [ ] AC-03: `FundOperationsPage` 에서 펀드 행을 클릭하면 해당 펀드의 상세 정보 페이지로 정상 이동(`useNavigate`)해야 한다.
- [ ] AC-04: `FundOperationsPage` 가 갖고 있던 기존 핵심 모듈 3대장(Transfer, Distribution, Assembly)이 개별 펀드의 `FundDetailPage.tsx` 탭 속으로 **로직이나 상태 유실 없이 완벽하게 이식·작동**해야 한다.
