# Phase 28: 조합 상세(FundDetailPage) 전체 UX 개편 및 LP-출자 통합 데이터 그리드 구축

> **Priority:** P0 (Core Fintech Functionality & Navigation UX Transformation)

---

## 🚨 치명적 핵심 원칙 (Non-negotiable Rules)
이번 페이즈의 목적은 기존 기능(백엔드API, 연동 상태)을 **단 1mg도 훼손하지 않은 채** 사용자의 눈에 보이는 "그릇(UI 레이아웃)"만 엑셀 형 모던 탭(Tab) 기반으로 바꾸는 것입니다.

1. **기능 무결성 유지 (Regression 제로):** `editingFund`, `editingLPId`, `editingNotices` 등의 인라인 폼 편집 기능, `queryClient.invalidateQueries`의 연쇄 동작, 그리고 기존 표들의 라우팅(마이크로 버튼, 워크플로 바로가기 등)은 탭 전환 컴포넌트 이사 시 절대 유실되지 않고 100% 동일하게 이관되어야 합니다.
2. **Zero Backend Modification:** 현업의 Excel 경험 및 모던 ERP Navigation(UX)만을 V:ON ERP 안에 병합하는 프론트엔드 리디자인 작업입니다. 새로운 백엔드 API, 쿼리문 변경, 데이터 구조 변형은 절대 금지합니다.
3. **`CapitalCallWizard` 완벽 재사용:** 현재 `FundDetailPage.tsx` 내에 일괄 출자요청(퍼센트/금액 기반) 모달 기능인 `CapitalCallWizard`가 이미 완벽히 구현되어 있습니다. 새로운 위저드 컴포넌트를 만드는 헛수고를 하지 말고, **통합표 상단 버튼 이벤트 트리거 방식을 통해 이식**만 하십시오.

## 현재 상태 및 목표

현재 조합 상세 페이지(`FundDetailPage.tsx`)는 10여 가지의 엄청난 양의 정보(기본정보, 워크플로 진행, 통지기간, 주요계약조항, LP목록, 미수서류, 투자목록 등)가 세로로 일렬로 끝없이 나열되어 있는 **무한 스크롤 형태(Infinite Scroll)**입니다. 이를 타파하고 **모던 Tab 내비게이션**을 도입하여, 가장 중요한 **자본 현황(LP 목록 + 출자 이력 통합 표)**을 엑셀(Data-Grid) 스타일로 재편성하는 목표를 갖습니다.

## Part 1 — 5-Tab 기반 모던 ERP 레이아웃 구축 (무한 스크롤 타파)

### 1-A. `FundDetailPage.tsx` 의 레이아웃 대수술
- 기존의 스택(`space-y-4`) 구조를 걷어내고, 화면 상단에 탭 버튼(Tab Headers) 네비게이션을 붙입니다.
- **상단 고정 요소:** `조합명칭 Header` 및 `요약 대시보드 3개 카드(총 약정/투자건수/투자금액 합계)`는 탭 선택과 무관하게 상단에 항상 고정(Sticky/Persistent)됩니다.

### 1-B. 탭별 기존 카드(컴포넌트) 이사 배치 규칙
현재 늘어서 있는 컴포넌트 JSX 코드들을 버리지 말고 아래 5개의 탭 방(Room)으로 각각 스위칭 렌더링되도록 온전하게 격리/배치하십시오.

| Tab Name | Tab ID | 배치할 타겟 카드 및 정보 영역 |
|---|---|---|
| **대시보드 (Overview)** | `overview` | ① 결성 진행 컨트롤 패널 (isFormingFund 일 때만 렌더링) <br> ② 미수 서류 카드 |
| **기본정보** | `info` | ① 14칸짜리 조합 상세 기본 정보 박스 뷰 (Pencil 눌러서 뜨는 수정 모드 폼 포함) |
| **자본 및 LP 현황** | `capital` | 👉 **[Part 2, 3] 의 엑셀형 출자통합 표 1개** (여기에 기존 LP목록과 출자이력, 출자위저드가 통폐합됨) |
| **투자 포트폴리오** | `portfolio`| ① 펀드 투자 내역 (목록/카드 형식 뷰 모드 전체 이관) |
| **규약 및 컴플라이언스** | `terms`| ① 주요 계약 조항(Key Terms) 수정 카드 <br> ② 통지기간(Notice Period) 수정 카드 |

## Part 2 — 조합별 '출자자(LP)' 탭과 '출자관리(Capital Call)'의 완전 통합 (Tab 3)

### 2-A. 통합 캡테이블(Capital Table) 신설
`[Tab 3. 자본 및 LP 현황]` 에는 기존처럼 기존 "LP 목록 박스"와 "출자 이력 뷰"을 분리하여 렌더링하지 않습니다. LP의 정보에 각자의 누적 출자 현황을 가로축에 Join 한 형태의 **Data-Grid** 만 단독으로 렌더링합니다.

**✅ 필수 테이블 구성 요건 (Mock-up)**

| 조합원(LP) 명 | 성격(구분) | 총 약정액 | 누적 납입액 | 지분율(%) | 비고 |
|--------------|------------|-----------|-------------|-----------|------|
| 트리거투자파트너스(유) | 공동업무집행 | 250,000,000 | 75,000,000 | 1.22% | GP |
| 파트너스(주)     | 공동업무집행  | 250,000,000 | 75,000,000 | 1.22% | GP |
| (주)라이프       | 유한책임    | 1,000,000,000| 300,000,000 | 4.88% | LP |
| ...           | ...        | ...       | ...         | ...       | ...  |
| **합계(Total)** |            | **20,500M**| **6,150M**  | **100%**  | |

- **계산식 고정:** `지분율 = (개별 LP 약정액 / 전체 총 약정액) * 100` (소수점 2자리 표기)
- **CSS 트릭:** 엑셀 창처럼 가로 컬럼이 넓어져 스크롤을 하더라도 좌측 <조합원 명> 열은 **틀 고정(Sticky)**되어 시야에서 벗어나면 안 됩니다. (`position: sticky`, 우측 스와이프 지원)
- **합계(Footer) 행 고정:** 표의 최하단에는 각 열(총 약정액 Sum, 누적납입액 Sum)의 통계와 지분율 합산인 100.00% 이 엑셀 Sum() 함수처럼 강제 렌더링되어야 합니다.

## Part 3 — 원클릭 일괄 출자 요청 (Bulk Capital Call) 생성 기능

### 3-A. 기존 출자요청 위저드(`CapitalCallWizard`)의 재배치 이식
- Part 2에서 만든 거대 `[출자자 및 자본 통합 엑셀 표]` **우측 상단 (Header)** 에 다음과 같은 2개의 Action 버튼(Primary/Secondary)을 배치합니다.
  1. `[+ 출자요청 위저드]` 
  2. `[+ LP 추가]`
- 이 버튼을 누르면 기존처럼 똑같이 `showCapCallWizard` (또는 `showCreateLP`) 상태가 `true` 로 켜지면서 로드가 완벽하게 마무리되어야 합니다.
- (위저드 자체의 지분율 %에 따른 각 LP별 출자요청액 계산 로직, 그리고 금액 강제 하드코딩 오버라이드 로직 등은 이미 완성되어 있으니 그대로 둡니다.)
