# PRD 04 — 조합(Fund) 관리

> **상위 문서:** [PRD_MASTER.md](./PRD_MASTER.md)
> **관련 페이지:** `/funds`, `/fund-overview`, `/funds/:id`, `/fund-operations`
> **상태:** 🟡 개발 중 | **최종 업데이트:** 2026-02-23

---

## 1. 기능 개요

| 항목 | 내용 |
|---|---|
| **목적** | VC가 운용하는 조합(Fund)의 전체 생애주기(결성 → 운용 → 청산)를 단일 시스템에서 관리 |
| **가정** | 1. 조합은 반드시 하나의 GP Entity에 연결된다. 2. 등록성립일은 최초 생성 후 수정 불가. |
| **범위 밖** | 금융감독원 전자공시 자동 제출, 세금 신고 연동 |
| **MVP** | 조합 기본정보 CRUD + LP 명부 일괄 등록 + 캐피탈콜 생성/납입 확인 + SSOT 동기화 |

## 3. SSOT (단일 진실 공급원) 규칙

- `Fund.paid_in` 값은 해당 조합의 `CapitalCallItem.paid = True` 항목의 `amount` 합계와 항상 일치해야 함
- LP 납입 확인 처리 시 시스템이 자동으로 `Fund.paid_in`을 재계산
