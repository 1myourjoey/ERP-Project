# PRD 07 — 자금 및 회계 (Capital & Accounting)

> **상위 문서:** [PRD_MASTER.md](./PRD_MASTER.md)
> **관련 페이지:** `/transactions`, `/accounting`
> **상태:** 🟡 개발 중 | **최종 업데이트:** 2026-02-23

---

## 1. 기능 개요

| 항목 | 내용 |
|---|---|
| **목적** | 조합의 자금 거래(출자/분배/운용비 등)를 기록하고 회계 원장을 관리. 조 단위 금액 오입력을 시스템이 방어 |
| **가정** | 1. 조합별 독립된 계좌를 기준으로 거래를 기록한다. 2. 모든 금액 입력란에는 한글 보조 텍스트가 실시간 표시된다. |
| **범위 밖** | 은행 API 연동 자동 거래 수집, 세무사 신고용 전자장부 export |
| **MVP** | 거래내역 CRUD + 회계 항목 전표 입력 + 한글 금액 보조 장치 + SSOT 동기화 |

---

## 2. 사용자 스토리 (User Stories)

### 자금 거래 내역 (`/transactions`)

| # | 제목 | 설명 | 우선순위 |
|---|---|---|---|
| US-01 | 거래내역 조회 | 관리자는 조합/기간 기준으로 자금 거래 내역(출자/분배/운용비 등)을 조회할 수 있다. | 필수 |
| US-02 | 거래 입력 | 관리자는 거래유형/금액/날짜/메모를 입력하여 거래를 기록할 수 있다. | 필수 |
| US-03 | 한글 금액 보조 | 금액 입력 시 하단에 "O억 O천만 원" 한글 단위가 실시간으로 표시되어 오입력을 방지한다. | 필수 |
| US-04 | SSOT 동기화 | 거래 저장 시 시스템이 조합의 재무 집계 수치(잔액 등)를 자동으로 갱신한다. | 필수 |
| US-05 | 거래 삭제 | 관리자는 잘못 입력된 거래를 삭제할 수 있다. | 필수 |

### 회계 원장 (`/accounting`)

| # | 제목 | 설명 | 우선순위 |
|---|---|---|---|
| US-06 | 원장 조회 | 관리자는 조합/기간/계정과목 기준으로 회계 원장을 조회할 수 있다. | 필수 |
| US-07 | 전표 입력 | 관리자는 계정과목/차변·대변/금액을 입력하여 회계 전표를 기록할 수 있다. | 필수 |
| US-08 | 임시저장 | 관리자는 작성 중인 전표를 임시저장하여 나중에 이어서 작성할 수 있다. | 선택 |

### Phase 33~35 상태 업데이트

| 대상 | 상태 | 변경 내용 |
|---|---|---|
| US-01 | ✅ 완료 | 거래원장 조회 API(`GET /api/transactions/ledger`)와 유형/기간/투자건 필터를 구현 |
| US-02 | ✅ 완료 | 거래 등록 시 유형별 동적 필드를 지원하도록 거래 모델/폼을 확장 |
| US-04 | ✅ 완료 | 거래 요약 API(`GET /api/transactions/summary`)와 대시보드 연계 집계를 추가 |
| US-NEW-09 | ✅ 완료 | 관리보수/성과보수 관리 기능(계산/설정/시뮬레이션/워터폴)을 신규 구현 |

## 3. 에러-프루프 (Error-Proof) 규칙

- **한글 금액 보조 (Korean Won Helper):** 모든 금액 Input에 숫자 입력 즉시 "O억 O천만 원" 파싱 텍스트 실시간 렌더링 (CurrencyInput 공통 컴포넌트 활용)
- **SSOT 무결성:** 거래 저장/삭제 후 시스템이 `Fund.paid_in` 및 관련 집계 수치를 자동 재계산
