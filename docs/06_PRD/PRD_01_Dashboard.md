# PRD 01 — 게이트웨이 대시보드

> **상위 문서:** [PRD_MASTER.md](./PRD_MASTER.md)
> **관련 페이지:** `/dashboard`
> **상태:** 🟡 개발 중 | **최종 업데이트:** 2026-02-24

---

## 1. 기능 개요

| 항목 | 내용 |
|---|---|
| **목적** | 1~2인 VC 관리팀이 출근 직후 30초 안에 오늘의 핵심 현황을 파악하는 경량 허브 |
| **가정** | 사용자는 대시보드에서 깊은 업무 처리를 원하지 않는다. 클릭 → 해당 페이지로 즉시 이동. |
| **범위 밖** | 대시보드 내 직접 Task 생성/편집, 인라인 데이터 수정 |
| **MVP** | KPI 요약 위젯(AUM/마감건수/미납 캐피탈콜) + 긴급 Task 알림 + 빠른 이동 링크 |

---

## 2. 사용자 스토리 (User Stories)

| # | 제목 | 설명 | 우선순위 | 상태 |
|---|---|---|---|---|
| US-01 | KPI 위젯 조회 | 관리자는 전체 AUM, 운용 중인 조합 수, 이번 주 마감 Task 수를 한눈에 볼 수 있다. | 필수 | ✅ 완료 |
| US-02 | 긴급 Task 알림 | 관리자는 D-Day가 오늘이거나 지난 Task를 대시보드에서 즉시 확인하고 업무보드로 이동할 수 있다. (오늘 업무 카드에 지연 건수 병기) | 필수 | 🔄 변경됨 |
| US-03 | 캐피탈콜 미납 알림 | 관리자는 납입 기일이 경과했으나 미납된 LP 현황을 대시보드에서 볼 수 있다. | 필수 | ✅ 완료 |
| US-04 | 빠른 이동 | 관리자는 위젯/패널 CTA 클릭 한 번으로 업무보드/업무 파이프라인/펀드상세/LP관리 등 해당 페이지로 즉시 이동할 수 있다. | 필수 | 🔄 변경됨 |
| US-05 | 주간 캘린더 미리보기 | 관리자는 이번 주 예정된 이벤트(캐피탈콜, 총회 등)를 대시보드에서 미리볼 수 있다. | 선택 | ✅ 완료 |
| US-06 | 통지/보고 출처 통합 | 관리자는 우측 패널 통지/보고 항목에서 `[조합규약]`/`[워크플로]`/`[업무]` 출처를 구분해 확인하고 해당 화면으로 즉시 이동할 수 있다. | 필수 | ✅ 완료 |
| US-07 | 통지/보고 업무 연계 | 관리자는 통지/보고 패널에서 워크플로 단계뿐 아니라 `is_notice`/`is_report`가 지정된 업무(Task)도 동일 규칙으로 확인할 수 있으며, 정기업무 관련 항목은 `정기` 배지로 구분할 수 있다. | 필수 | 🔄 변경됨 |
| US-08 | 정기업무 긴급 배너 | 관리자는 정기업무(Task)의 임박/지연 항목을 상단 긴급 배너에서 확인하고 업무보드로 즉시 이동할 수 있다. | 필수 | ✅ 완료 |
| US-09 | 투자/재무 확장 위젯 | 관리자는 대시보드에서 진행 심의 건수, 운용 NAV, 미수령 보수, 진행 보고 건수를 즉시 확인할 수 있다. | 필수 | ✅ 완료 |
| US-10 | 확장 위젯 라우팅 | 관리자는 확장 위젯 클릭으로 투자 심의/가치평가/보수 관리/영업보고 화면으로 즉시 이동할 수 있다. | 필수 | ✅ 완료 |
| US-11 | 컴플라이언스 위젯 | 관리자는 대시보드에서 기한초과/이번주 마감 컴플라이언스 건수를 확인하고 컴플라이언스 화면으로 이동할 수 있다. | 필수 | ✅ 완료 |
| US-NEW-12 | 서류 수집 위젯 | 관리자는 대시보드에서 현재 분기 서류 수집률과 미수집 기업 수를 확인하고 영업보고 화면으로 이동할 수 있다. | 필수 | ✅ 완료 |
| US-NEW-13 | 긴급 알림 통합 | 관리자는 컴플라이언스 연체/수시보고/내부보고회 지연 이슈를 하나의 긴급 알림 영역에서 확인할 수 있다. | 필수 | ✅ 완료 |
| US-NEW-14 | 조합별 Quick View 확장 | 관리자는 조합 Quick View에서 컴플라이언스 지연 건수와 서류 수집 진행률을 함께 확인할 수 있다. | 필수 | ✅ 완료 |
| US-NEW-15 | 오늘의 우선순위 업무 리스트 | 관리자는 미완료 업무를 기한초과→D-Day→워크플로 단계 기준으로 정렬한 우선순위 리스트(상위 10건 + 나머지 보기)로 확인할 수 있다. | 필수 | ✅ 완료 |

---

## 3. 비기능 요구사항

- 첫 화면 로딩 시 모든 위젯 데이터를 **병렬(Parallel) API 호출**로 가져와 지연 없이 렌더링
- 대시보드 위젯은 **읽기 전용(Read-only)**, 수정 UI 일절 없음
