# PRD 06 — 투자/포트폴리오 관리 (Investments)

> **상위 문서:** [PRD_MASTER.md](./PRD_MASTER.md)
> **관련 페이지:** `/investments`, `/investments/:id`, `/valuations`, `/exits`
> **상태:** 🟡 개발 중 | **최종 업데이트:** 2026-02-23

---

## 1. 기능 개요

| 항목 | 내용 |
|---|---|
| **목적** | 조합에서 집행된 투자의 포트폴리오를 관리하고, 주기적 가치평가(Valuation)와 최종 회수(Exit)를 통해 수익률을 트래킹 |
| **가정** | 1. 하나의 기업에 대해 여러 라운드 투자가 가능하다. 2. 가치평가는 반기/연간 주기로 수행된다. |
| **범위 밖** | 실시간 주가 데이터 연동, DART 전자공시 파싱 |
| **MVP** | 투자 기업 CRUD + 라운드별 투자 내역 + 가치평가 기록 + Exit 처리 |

---

## 2. 사용자 스토리 (User Stories)

### 투자 목록 (`/investments`)

| # | 제목 | 설명 | 우선순위 |
|---|---|---|---|
| US-01 | 투자 목록 조회 | 관리자는 전체 포트폴리오 기업 목록(기업명/투자일/금액/상태)을 조회할 수 있다. | 필수 |
| US-02 | 투자 등록 | 관리자는 기업명/투자일/금액/지분/연결 조합을 입력하여 신규 투자를 등록할 수 있다. | 필수 |

### 투자 상세 (`/investments/:id`)

| # | 제목 | 설명 | 우선순위 |
|---|---|---|---|
| US-03 | 기업 기본정보 편집 | 관리자는 기업의 업종/대표/연락처 등 기본정보를 편집할 수 있다. | 필수 |
| US-04 | 라운드별 투자 내역 | 관리자는 시리즈 A/B 등 라운드별 투자 금액과 지분율을 기록할 수 있다. | 필수 |
| US-05 | Multiple/IRR 자동 계산 | 시스템은 가치평가액과 투자원금을 기반으로 Multiple과 IRR을 자동 계산하여 표시한다. | 필수 |

### 가치평가 (`/valuations`)

| # | 제목 | 설명 | 우선순위 |
|---|---|---|---|
| US-06 | 가치평가 기록 | 관리자는 기준일/기업/평가액을 입력하여 공정가치 평가를 기록할 수 있다. | 필수 |
| US-07 | 전체 조회 | 관리자는 조합/기간 기준으로 전체 가치평가 내역을 조회할 수 있다. | 필수 |

### Exit 처리 (`/exits`)

| # | 제목 | 설명 | 우선순위 |
|---|---|---|---|
| US-08 | Exit 등록 | 관리자는 Exit일/회수액/매각 주식 수를 입력하여 투자 회수 이벤트를 기록할 수 있다. | 필수 |
| US-09 | MoIC 자동 계산 | 시스템은 Exit 처리 후 해당 투자의 MoIC(투자금 대비 회수배수)를 자동 계산한다. | 필수 |

### Phase 33~35 상태 업데이트

| 대상 | 상태 | 변경 내용 |
|---|---|---|
| US-01 | ✅ 완료 | 투자 심의 파이프라인(`investment_reviews`)과 투자 전환(convert) 연계를 구현 |
| US-03 | ✅ 완료 | 투자 상세에 심의 이력(코멘트 타임라인) 연계 표시를 추가 |
| US-06 | ✅ 완료 | 가치평가 모델 확장(방법론/수단/공정가치/미실현손익) 및 대시보드/이력/일괄평가를 구현 |
| US-07 | ✅ 완료 | 조합별 NAV 요약/대시보드 API 및 재원별 필터 조회를 구현 |
| US-08 | ✅ 완료 | 회수 거래 정산(`settle`) 처리와 거래원장 자동 등록을 구현 |
| US-09 | ✅ 완료 | 투자건별 MOIC 집계 대시보드 및 회수 정산 후 LP 배분 자동 생성을 구현 |
| US-NEW-10 | ✅ 완료 | 거래원장 세분화(`transaction_subtype`, `counterparty`, `conversion_detail`, `settlement_date`)와 ledger/summary API를 구현 |
