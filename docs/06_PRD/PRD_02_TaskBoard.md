# PRD 02 — 업무 보드 (Task Board & WorkLogs)

> **상위 문서:** [PRD_MASTER.md](./PRD_MASTER.md)
> **관련 페이지:** `/tasks`, `/worklogs`
> **상태:** 🟡 개발 중 | **최종 업데이트:** 2026-02-22

---

## 1. 기능 개요

| 항목 | 내용 |
|---|---|
| **목적** | VC 관리팀이 처리해야 할 모든 업무를 누락 없이 관리하고, 완료 시 휴먼 에러를 시스템이 차단하는 게이트키퍼 역할 |
| **가정** | 1. 사용자는 Q1~Q4 아이젠하워 매트릭스 방식에 익숙하다. 2. 모든 Task는 마감일(deadline)을 보유한다. |
| **범위 밖** | 외부 캘린더(Google/Outlook) 동기화, 모바일 푸시 알림 |
| **MVP** | Q1~Q4 칸반 뷰 + Task CRUD + 완료 시 실제시간 기록 + 업무일지(WorkLog) 연동 |

---

## 2. 사용자 스토리 (User Stories)

| # | 제목 | 설명 | 우선순위 | 상태 |
|---|---|---|---|---|
| US-01 | Task 생성 | 관리자는 제목/마감일/우선순위/연결 조합/카테고리를 입력하여 Task를 생성할 수 있다. | 필수 | 🔄 변경됨 |
| US-02 | Q1~Q4 칸반 조회 | 관리자는 긴급·중요도에 따라 분류된 4개 칸반 보드에서 전체 업무를 조회할 수 있다. | 필수 | ✅ 완료 |
| US-03 | Task 완료 처리 | 관리자는 Task 완료 시 실제 소요시간을 기입해야 하며, 워크플로 필수 서류 미충족 시 완료가 잠금된다. 완료 모달에서 동일 카테고리의 과거 교훈을 함께 확인한다. | 필수 | 🔄 변경됨 |
| US-04 | WorkLog 자동 생성 | 관리자가 Task를 완료할 때 업무 일지(WorkLog)를 자동 생성할지 선택할 수 있다. | 필수 | ✅ 완료 |
| US-05 | Q간 드래그 이동 | 관리자는 Task를 드래그하여 Q1~Q4 사이를 이동시킬 수 있다. | 필수 | ✅ 완료 |
| US-06 | 워크플로 연동 Task | 관리자는 워크플로 단계와 연결된 Task를 업무보드에서 그룹으로 확인할 수 있다. | 필수 | ✅ 완료 |
| US-07 | Task 삭제 | 관리자는 불필요한 Task를 삭제할 수 있다. (완료된 Task는 WorkLog에 보존) | 필수 | 🔄 변경됨 |
| US-08 | 펀드/GP 필터 | 관리자는 특정 조합이나 GP 기준으로 Task를 필터링하여 볼 수 있다. | 선택 | ✅ 완료 |
| US-09 | WorkLog 수동 추가 | 관리자는 Task 없이도 업무 일지를 수동으로 추가할 수 있다. | 선택 | 🟡 진행 중 |
| US-10 | D-Day 색상 경고 | 관리자는 Task 카드/캘린더 셀·dot/파이프라인 카드에서 지연·오늘·임박 업무를 즉시 구분할 수 있다. (지연/오늘 색 대비 강화) | 필수 | 🔄 변경됨 |
| US-11 | 일괄 처리(Bulk) | 관리자는 체크박스로 여러 Task를 선택해 일괄 완료/삭제할 수 있다. | 필수 | ✅ 완료 |
| US-12 | 지연/임박 경고 배너 | 관리자는 업무보드 상단에서 지연 업무(빨강)와 24시간 이내 임박 업무(주황)를 분리된 배너로 확인하고, `[업무 확인]` 클릭 시 해당 Task로 스크롤+하이라이트 이동할 수 있다. | 필수 | 🔄 변경됨 |
| US-13 | 필터 의미 안내 | 관리자는 `[진행 중 | 전체 | 완료]` 탭의 툴팁으로 각 필터의 기준을 즉시 이해할 수 있다. | 필수 | ✅ 완료 |
| US-14 | 캘린더 빠른 완료 | 관리자는 캘린더 이벤트 목록의 Task 항목에서 ✅ 버튼으로 CompleteModal을 열어 빠르게 완료 처리할 수 있다. | 필수 | ✅ 완료 |
| US-15 | 파이프라인 보조 뷰 | 관리자는 업무보드 내 3탭(`[보드 | 캘린더 | 파이프라인]`)으로 칸반·캘린더·파이프라인을 전환해 본다. 파이프라인은 상단 대기(기한 미배정)+하단 4열 구조로 표시되며, 대기 0건 시 접힘 요약 상태로 표시된다. | 필수 | 🔄 변경됨 |
| US-16 | 카테고리 관리 | 관리자는 업무보드에서 카테고리를 추가/삭제하고, 사용 중인 카테고리 삭제 시 경고를 확인할 수 있다. | 필수 | ✅ 완료 |
| US-17 | 카테고리 기반 교훈 리마인드 | 관리자는 업무 완료 시 CompleteModal에서 동일 카테고리의 과거 교훈(최대 5건)을 확인할 수 있다. | 필수 | ✅ 완료 |
| US-18 | 카테고리 정합성 자동등록 | 관리자는 워크플로/업무/업무일지 생성 시 카테고리가 task_categories와 자동 동기화되어 교훈 리마인드가 끊기지 않는다. | 필수 | ✅ 완료 |

## 3. 에러-프루프 (Error-Proof) 규칙

> Phase 31 핵심 기획 내용

- **완료 Lock 조건:** 실제 소요시간(actual_time) 미기입 시 완료 버튼 비활성화
- **워크플로 연동 Task Lock:** 해당 단계의 필수 서류가 미첨부된 경우 완료 처리 거절 + 에러 툴팁
- **캐피탈콜 경고:** 투자실행/캐피탈콜 관련 Task 완료 시 연결 Fund의 최근 CapitalCall 금액·기일 누락 경고 표시 (완료는 허용)
