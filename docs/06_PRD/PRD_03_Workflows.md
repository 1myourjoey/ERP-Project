> **관련 페이지:** `/workflows`
> **상태:** 🟡 개발 중 | **최종 업데이트:** 2026-02-22

---

## 1. 기능 개요

| 항목 | 내용 |
|---|---|
| **목적** | 결성/청산/캐피탈콜 등 반복적인 업무 절차(SOP)를 표준 템플릿으로 정의하고, 조합에 인스턴스를 생성하여 단계별로 관리 |
| **가정** | 1. 각 워크플로는 여러 단계(Steps)로 구성된다. 2. 단계마다 필수 서류가 존재할 수 있다. |
| **범위 밖** | 외부 전자서명(DocuSign 등) 연동, 자동 이메일 발송 |
| **MVP** | 템플릿 CRUD + 조합에 워크플로 인스턴스 생성 + 단계별 완료 처리(서류 검증 포함) |

---

## 2. 사용자 스토리 (User Stories)

| # | 제목 | 설명 | 우선순위 | 상태 |
|---|---|---|---|---|
| US-01 | 템플릿 조회 | 관리자는 등록된 모든 워크플로 템플릿 목록을 조회할 수 있다. | 필수 | ✅ 완료 |
| US-02 | 템플릿 생성 | 관리자는 이름/카테고리/단계/필수서류를 지정하여 새로운 워크플로 템플릿을 생성할 수 있다. | 필수 | ✅ 완료 |
| US-03 | 인스턴스 생성 | 관리자는 특정 조합에 워크플로 템플릿을 적용하여 인스턴스를 생성할 수 있다. | 필수 | ✅ 완료 |
| US-04 | 단계 완료 처리 | 관리자는 각 단계를 완료 처리할 수 있으며, 필수 서류 미확인 항목이 있으면 경고 후 확인 시 완료할 수 있다. 단계 완료 시 WorkLog가 자동 기록된다. | 필수 | 🔄 변경됨 |
| US-05 | 진행률 표시 | 관리자는 워크플로 인스턴스 카드에서 진행률 바/현재 단계 강조/D-Day 배지/임박순 정렬/요약 배너를 확인할 수 있으며, 각 단계별 서류 체크리스트(체크/추가/수정/삭제)를 인스턴스 단위로 관리할 수 있다. | 필수 | 🔄 변경됨 |
| US-06 | 인스턴스 취소 | 관리자는 진행 중인 워크플로 인스턴스를 취소할 수 있다. | 필수 | ✅ 완료 |
| US-07 | 체크리스트 인쇄 | 관리자는 워크플로 인스턴스의 체크리스트를 프린트할 수 있다. | 선택 | 🟡 진행 중 |
| US-08 | 통지 유형 선택 | 관리자는 워크플로 단계에 통지 유형(총회/캐피탈콜/분배 등)을 연결할 수 있고, UI에서는 한글 표시명만 확인한다. | 선택 | 🔄 변경됨 |
| US-09 | 레거시 체크리스트 통합 | 관리자는 워크플로 페이지 내 `체크리스트(레거시)` 탭에서 기존 체크리스트를 조회하고 워크플로 템플릿으로 변환할 수 있다. | 필수 | ✅ 완료 |

---

## 3. 에러-프루프 (Error-Proof) 규칙

- **단계 완료 경고 조건:** 해당 단계에 필수 서류(required document)가 미확인 상태이면 완료 전 확인 경고를 표시하고, 사용자 재확인 후 완료를 진행한다.
